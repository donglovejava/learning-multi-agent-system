## 1 引言

### 1.1 编写目的

本文档是"基于大模型的个性化资源生成与学习多智能体系统"（以下简称"本系统"）的软件设计说明书（Software Design Document, SDD）V5.0 版本。

**V5.0 版本重点：**
- 突出 3 大核心创新的技术实现（可解释 AI、遗忘曲线、脚手架）
- 每个模块明确输入/输出/流程
- 保留 V3 的详细设计深度，每个 Agent 设计精确到可直接编码实现
- 增加完整的代码示例、架构图、流程图

**文档用途：**

1. **编码实现指南**：为每位开发工程师提供其所负责模块的详细设计规格，包括数据结构、算法伪代码、接口定义、状态机转换等，使开发人员无需额外沟通即可开始编码。
2. **竞赛评审材料**：作为中国软件杯 A3 赛题的核心技术文档，向评审专家展示系统的技术深度、架构合理性和创新价值。
3. **测试基准文档**：为测试工程师提供测试用例设计的依据，包括每个 Agent 的输入输出规范、异常场景、性能指标等。
4. **运维部署手册**：为运维人员提供完整的部署配置、监控指标、告警策略、扩展方案等。

**读者对象：**

| 读者角色 | 关注章节 | 使用方式 |
|----------|----------|----------|
| 架构师 | 第 2 章（总体设计）、第 8 章（部署设计） | 全局把控架构合理性 |
| 后端开发工程师 | 第 3 章（核心创新）、第 4 章（Agent 设计）、第 5 章（接口）、第 6 章（数据结构） | 按模块编码实现 |
| 前端开发工程师 | 第 5.1 节（用户接口）、第 5.3 节（内部接口） | 页面开发与 API 对接 |
| Agent 开发工程师 | 第 4.2 节（Agent 详细设计）、附录 C（Prompt 模板） | Agent 编码与调试 |
| 测试工程师 | 第 4.2 节（Agent 测试用例）、第 5 章（接口定义） | 测试用例设计 |
| 运维工程师 | 第 8 章（部署设计） | 环境部署与运维 |
| 竞赛评审专家 | 第 3 章（核心创新）、第 2 章（架构设计） | 技术方案评审 |

### 1.2 项目背景

#### 1.2.1 一句话说明

**我们构建了一个由 11 个 AI 智能体协同工作的学习系统——它通过对话了解学生，30 秒内生成 6 种个性化学习资料，并持续跟踪学习效果、动态调整学习路径。**

#### 1.2.2 核心功能说明（用具体场景）

**场景 1：大一学生小李想学"机器学习"**

1. 小李登录系统，系统通过对话了解他是计算机专业大二学生，目标是准备考研
2. 系统自动评估他的数学和编程基础，发现他线性代数不错但概率论薄弱
3. 系统为他生成：
   - 一份 2000 字的"机器学习概述"讲解文档（难度适配他的水平）
   - 10 道练习题（4 道基础、4 道中等、2 道提高）
   - 一张机器学习知识体系思维导图
   - 一个"线性回归原理"的 90 秒动画视频
4. 系统为他规划了一条 8 周的学习路径，从基础概念到算法实现

**场景 2：小李学了一周后**

1. 系统检测到他在"决策树"章节的练习得分只有 55%
2. 系统自动触发预警，生成一份补充讲解文档
3. 系统调整学习路径，在后续安排中增加了决策树的复习节点
4. 系统推送 3 道针对性的练习题帮助巩固

**场景 3：教师王老师上传课程资料**

1. 王老师上传了"人工智能导论"的 PPT 和教材
2. 系统自动解析文档，构建包含 150 个知识点的知识图谱
3. 系统自动生成 50 道练习题和 10 份讲解文档
4. 王老师审核并修改后发布给学生使用

#### 1.2.3 用户价值说明（用数据说话）

| 价值维度 | 传统方式 | 本系统 | 提升幅度 |
|----------|----------|--------|----------|
| 资源获取时间 | 平均 30 分钟搜索和整理 | 平均 30 秒自动生成 | **60 倍提升** |
| 个性化程度 | 统一教材，无个性化 | 6 维度画像，完全个性化 | **从 0 到 1** |
| 教师备课时间 | 每周 8-12 小时 | 每周 4-6 小时（审核为主） | **减少 40-50%** |
| 学习效率 | baseline | 提升 20-30% | **显著提升** |
| 资源类型 | 文本 + 视频 | 6 种多模态资源 | **3 倍丰富** |

### 1.3 术语定义

| 序号 | 术语/缩略语 | 英文全称 | 详细定义 | 使用场景示例 |
|------|-------------|----------|----------|-------------|
| 1 | LLM | Large Language Model | 大语言模型，指具有大规模参数量的预训练语言模型 | "系统使用讯飞星火 LLM 作为核心推理引擎" |
| 2 | RAG | Retrieval-Augmented Generation | 检索增强生成，一种结合知识检索与模型生成的技术范式 | "RAG 模块从知识库中检索相关知识点后再生成讲解文档" |
| 3 | SeeDance | - | 科大讯飞多模态生成大模型，支持视频/动画等内容的智能生成 | "Video Agent 调用 SeeDance API 生成教学动画" |
| 4 | Multi-Agent | Multi-Agent System | 多智能体系统，由多个具有不同角色和能力的 AI 智能体协同工作的系统架构 | "本系统采用 11 个 Agent 的 Multi-Agent 架构" |
| 5 | Agent | Intelligent Agent | 智能体，具备自主感知、决策、执行能力的 AI 程序实体 | "Profile Agent 负责构建学生学习画像" |
| 6 | Orchestrator | Orchestrator Agent | 编排器 Agent，负责整体任务调度、流程编排和 Agent 间协调 | "Orchestrator 接收用户请求后分解任务并分配给各 Agent" |
| 7 | 学习画像 | Learner Profile | 基于多维度数据构建的学习者特征模型 | "系统为每位学生构建 6 维度的动态学习画像" |
| 8 | 知识图谱 | Knowledge Graph | 以图结构组织知识实体及其关系的语义网络 | "知识图谱存储知识点之间的前置/后续关系" |
| 9 | DAG | Directed Acyclic Graph | 有向无环图，用于建模知识点之间的依赖关系和学习路径 | "学习路径基于 DAG 建模，节点为知识点，边为学习关系" |
| 10 | EWMA | Exponentially Weighted Moving Average | 指数加权移动平均，一种时间序列平滑算法，用于动态更新画像维度值 | "画像维度值使用 EWMA 算法动态更新" |
| 11 | LangGraph | - | 多智能体编排框架，支持图编排、状态管理、人机协同 | "基于 LangGraph 实现 Agent 图编排" |
| 12 | RRF | Reciprocal Rank Fusion | 融合排序算法，用于合并多个检索结果 | "使用 RRF 算法融合向量和图谱检索结果" |
| 13 | HNSW | Hierarchical Navigable Small World | 近似最近邻搜索算法，用于向量数据库 | "Milvus 使用 HNSW 索引加速向量检索" |
| 14 | JWT | JSON Web Token | 一种用于身份验证的令牌标准 | "用户使用 JWT Token 进行身份验证" |
| 15 | SSE | Server-Sent Events | 服务器推送技术，用于流式输出 | "对话界面使用 SSE 实现流式输出" |

#### 2.1.2 架构设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **模块化** | 每个 Agent 独立模块，可单独开发测试 | 每个 Agent 一个 Python 类 |
| **松耦合** | Agent 间通过消息队列通信，不直接调用 | RabbitMQ 异步消息 |
| **高可用** | 单个 Agent 故障不影响整体系统 | 故障隔离 + 降级运行 |
| **可扩展** | 新增 Agent 不影响现有架构 | LangGraph 图编排 |
| **可观测** | 每个 Agent 的执行状态可追踪 | 日志 + 监控面板 |

### 3.1 创新一：可解释 AI 决策

#### 3.1.1 问题定义

**现有问题：**
- AI 推荐学习内容，但不解释为什么
- 学生不知道为什么要学这个，学习动机不足
- 路径调整没有理由，学生不理解
- 教师无法审核 AI 决策的合理性

**目标：** 每个推荐都有理由，学生知道为什么学这个，教师可以审核决策依据。

#### 3.1.2 技术方案

**核心思路：** 知识图谱推理 + 自然语言生成

```
用户请求："我想学 Transformer"
         │
         ▼
    ┌─────────────┐
    │ 查询知识图谱  │ ← 找到 Transformer 的前置知识链
    │ (Neo4j)      │   例：线性代数 → 概率论 → 注意力机制 → Transformer
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ 查询学生画像  │ ← 获取学生的知识掌握度
    │ (PostgreSQL) │   例：线性代数 85%, 概率论 45%, 注意力机制 0%
    └──────┬──────┘
           │
           ▼
    ┌─────────────
    │ 推理决策理由  │ ← 结合知识链和画像生成解释
    │              │   发现概率论掌握度 45% < 60% 阈值
    ──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ 生成自然语言  │ ← "推荐你先学概率论，因为你的概率论基础薄弱
    │ (LLM)        │   (掌握度 45%)，而它是 Transformer 的前置知识"
    ─────────────┘
```

#### 3.1.3 完整代码实现

```python
# core/explainable_recommender.py

from typing import List, Dict, Optional
from dataclasses import dataclass
from neo4j import GraphDatabase
import logging

logger = logging.getLogger(__name__)

@dataclass
class KnowledgeNode:
    """知识图谱节点"""
    name: str
    difficulty: int  # 1-5
    category: str
    mastery: float = 0.0  # 学生掌握度 0-1

@dataclass
class RecommendationResult:
    """推荐结果"""
    recommendation: str  # 推荐的知识点
    explanation: str  # 自然语言解释
    confidence: float  # 置信度 0-1
    reasoning_chain: List[str]  # 推理链
    weak_points: List[Dict]  # 薄弱环节

class ExplainableRecommender:
    """可解释的推荐系统"""
    
    def __init__(self, neo4j_uri: str, neo4j_user: str, neo4j_password: str):
        self.driver = GraphDatabase.driver(neo4j_uri, auth=(neo4j_user, neo4j_password))
        self.weak_threshold = 0.6  # 掌握度低于 60% 视为薄弱
    
    def recommend_with_explanation(
        self, 
        student_id: str, 
        target_knowledge: str,
        profile: Dict[str, float]
    ) -> RecommendationResult:
        """
        生成带解释的推荐
        
        Args:
            student_id: 学生 ID
            target_knowledge: 目标知识点名称
            profile: 学生画像 {知识点名称：掌握度}
        
        Returns:
            RecommendationResult 包含推荐、解释、置信度、推理链
        """
        # 1. 查询知识图谱，获取前置知识链
        knowledge_chain = self._query_prerequisite_chain(target_knowledge)
        
        if not knowledge_chain:
            return RecommendationResult(
                recommendation=target_knowledge,
                explanation=f"未找到{target_knowledge}的前置知识，可以直接学习",
                confidence=0.5,
                reasoning_chain=[],
                weak_points=[]
            )
        
        # 2. 找出知识链中的薄弱环节
        weak_points = self._find_weak_points(knowledge_chain, profile)
        
        # 3. 生成推荐和解释
        if not weak_points:
            recommendation = target_knowledge
            explanation = (
                f"推荐你直接学习{target_knowledge}，"
                f"因为你已经掌握了所有前置知识。"
            )
            confidence = 0.9
        else:
            weakest = weak_points[0]
            recommendation = weakest['name']
            explanation = self._generate_explanation(
                target=target_knowledge,
                weak_point=weakest,
                chain_length=len(knowledge_chain)
            )
            confidence = self._calc_confidence(weak_points)
        
        reasoning_chain = [node['name'] for node in knowledge_chain]
        
        return RecommendationResult(
            recommendation=recommendation,
            explanation=explanation,
            confidence=confidence,
            reasoning_chain=reasoning_chain,
            weak_points=weak_points
        )
    
    def _query_prerequisite_chain(self, target: str) -> List[Dict]:
        """查询知识图谱，获取目标知识点的前置知识链"""
        query = """
        MATCH path = (a:Knowledge)-[:PREREQUISITE*]->(b:Knowledge {name: $target})
        WITH nodes(path) AS nodes
        UNWIND nodes AS node
        RETURN node.name AS name, 
               node.difficulty AS difficulty,
               node.category AS category
        ORDER BY length(path) DESC
        """
        
        with self.driver.session() as session:
            result = session.run(query, target=target)
            return [dict(record) for record in result]
    
    def _find_weak_points(
        self, 
        chain: List[Dict], 
        profile: Dict[str, float]
    ) -> List[Dict]:
        """找出知识链中的薄弱环节"""
        weak = []
        for node in chain:
            name = node['name']
            mastery = profile.get(name, 0.5)
            
            if mastery < self.weak_threshold:
                weak.append({
                    'name': name,
                    'mastery': mastery,
                    'difficulty': node['difficulty'],
                    'category': node['category']
                })
        
        weak.sort(key=lambda x: x['mastery'])
        return weak
    
    def _generate_explanation(
        self, 
        target: str, 
        weak_point: Dict, 
        chain_length: int
    ) -> str:
        """生成自然语言解释"""
        name = weak_point['name']
        mastery_pct = weak_point['mastery'] * 100
        
        explanation = (
            f"推荐你先学{name}，"
            f"因为你的{name}基础薄弱（掌握度{mastery_pct:.0f}%），"
            f"而它是学习{target}的必要前置知识。"
        )
        
        if chain_length > 2:
            explanation += f"掌握{name}后，后续学习会更顺利。"
        
        difficulty = weak_point['difficulty']
        if difficulty >= 4:
            explanation += f"这个知识点难度较高，建议多花时间理解。"
        elif difficulty <= 2:
            explanation += f"这个知识点相对基础，可以快速掌握。"
        
        return explanation
    
    def _calc_confidence(self, weak_points: List[Dict]) -> float:
        """计算推荐置信度"""
        if not weak_points:
            return 0.9
        
        avg_mastery = sum(wp['mastery'] for wp in weak_points) / len(weak_points)
        confidence = min(0.5 + avg_mastery * 0.5, 0.95)
        
        return confidence
    
    def explain_path_adjustment(
        self,
        student_id: str,
        original_path: List[str],
        new_path: List[str],
        assessment_data: Dict
    ) -> str:
        """解释路径调整的理由"""
        added_nodes = set(new_path) - set(original_path)
        removed_nodes = set(original_path) - set(new_path)
        
        explanation_parts = []
        
        for node in added_nodes:
            score = assessment_data.get(node, {}).get('score', 0)
            if score < 0.6:
                explanation_parts.append(
                    f"检测到你在{node}章节得分{score*100:.0f}%，所以增加了针对性练习"
                )
        
        for node in removed_nodes:
            score = assessment_data.get(node, {}).get('score', 0)
            if score > 0.9:
                explanation_parts.append(
                    f"你在{node}章节表现优秀（{score*100:.0f}%），所以跳过了重复内容"
                )
        
        if not explanation_parts:
            return "根据最新学习数据微调了路径顺序"
        
        return "；".join(explanation_parts)
    
    def close(self):
        """关闭 Neo4j 连接"""
        self.driver.close()
```

### 3.2 创新二：个性化遗忘曲线

#### 3.2.1 问题定义

**现有问题：**
- 统一复习计划（1 天后、3 天后、7 天后）
- 不考虑个体差异（有人记得快，有人忘得快）
- 不考虑知识特性（概念性知识 vs 程序性知识遗忘速度不同）
- 无法量化"什么时候复习效果最好"

**目标：** 根据每个学生的实际测试数据，建立专属遗忘曲线，在最佳时间点推送复习。

#### 3.2.2 技术方案

**核心思路：** 指数衰减模型拟合 + 最优复习时间计算

```
学生学习知识点 A
    │
    ├── 记录学习时间 t0 和初始测试得分 s0 (假设 s0=0.95)
    │
    ▼
学生在 t1, t2, t3... 时刻复习
    │
    ├── 记录每次复习的得分：
    │   t1=1 天，s1=0.85
    │   t2=3 天，s2=0.70
    │   t3=7 天，s3=0.50
    │
    ▼
用指数模型拟合：s = a × exp(-b × t)
    │
    ├── 拟合结果：a=0.95, b=0.15
    │
    ▼
计算最佳复习时间：s 降到 70% 时的 t
    │
    ├── t_optimal = -ln(0.7/0.95) / 0.15 ≈ 2 天
    │
    ▼
生成个性化复习计划：每 2 天复习一次
```

#### 3.2.3 完整代码实现

```python
# core/forgetting_curve_model.py

import math
from datetime import datetime, timedelta
from typing import Dict, List, Tuple, Optional
from dataclasses import dataclass
import logging

logger = logging.getLogger(__name__)

@dataclass
class ReviewRecord:
    """复习记录"""
    knowledge_id: str
    time_diff_days: float  # 距上次学习的时间（天）
    score: float  # 测试得分 0-1
    review_type: str  # 'test', 'practice', 'self_assessment'

@dataclass
class ForgettingCurve:
    """遗忘曲线参数"""
    a: float  # 初始记忆强度
    b: float  # 遗忘率（越大忘得越快）
    optimal_interval: float  # 最佳复习间隔（天）
    data_points: int  # 数据点数量
    r_squared: float  # 拟合优度

@dataclass
class ReviewSchedule:
    """复习计划"""
    next_review: datetime
    optimal_interval: float
    predicted_score: float
    confidence: float

class ForgettingCurveModel:
    """个性化遗忘曲线模型"""
    
    def __init__(self, db_connection):
        self.db = db_connection
        self.curves: Dict[str, Dict[str, ForgettingCurve]] = {}
        self.target_score = 0.7  # 目标保持分数
        self.min_data_points = 3  # 最少需要 3 个数据点才能拟合
    
    def fit(
        self, 
        student_id: str, 
        review_history: Dict[str, List[ReviewRecord]]
    ) -> Dict[str, ForgettingCurve]:
        """
        用指数衰减模型拟合遗忘曲线
        
        Args:
            student_id: 学生 ID
            review_history: {
                "knowledge_id": [
                    ReviewRecord(time_diff_days=1, score=0.85),
                    ReviewRecord(time_diff_days=3, score=0.70),
                    ...
                ]
            }
        
        Returns:
            {knowledge_id: ForgettingCurve}
        """
        student_curves = {}
        
        for knowledge_id, records in review_history.items():
            if len(records) < self.min_data_points:
                logger.info(f"数据点不足，使用默认曲线")
                student_curves[knowledge_id] = self._default_curve()
                continue
            
            times = [r.time_diff_days for r in records]
            scores = [r.score for r in records]
            
            a, b, r_squared = self._fit_exponential(times, scores)
            optimal_time = self._calc_optimal_time(a, b)
            
            curve = ForgettingCurve(
                a=a, b=b, optimal_interval=optimal_time,
                data_points=len(records), r_squared=r_squared
            )
            
            student_curves[knowledge_id] = curve
            logger.info(
                f"学生 {student_id} 知识点 {knowledge_id} 遗忘曲线："
                f"a={a:.3f}, b={b:.3f}, 最佳间隔={optimal_time:.1f}天"
            )
        
        self.curves[student_id] = student_curves
        self._save_curves_to_db(student_id, student_curves)
        
        return student_curves
    
    def get_review_schedule(
        self, 
        student_id: str, 
        knowledge_id: str, 
        last_review_time: datetime
    ) -> ReviewSchedule:
        """获取个性化复习计划"""
        curve = self._get_curve(student_id, knowledge_id)
        
        if curve is None:
            return self._default_schedule(last_review_time)
        
        next_review = last_review_time + timedelta(days=curve.optimal_interval)
        predicted_score = self.target_score
        confidence = min(curve.r_squared * (curve.data_points / 5), 0.95)
        
        return ReviewSchedule(
            next_review=next_review,
            optimal_interval=curve.optimal_interval,
            predicted_score=predicted_score,
            confidence=confidence
        )
    
    def predict_score(
        self, 
        student_id: str, 
        knowledge_id: str, 
        days_since_review: float
    ) -> float:
        """预测指定天数后的记忆保持分数"""
        curve = self._get_curve(student_id, knowledge_id)
        
        if curve is None:
            return math.exp(-0.15 * days_since_review)
        
        predicted = curve.a * math.exp(-curve.b * days_since_review)
        return max(0.0, min(1.0, predicted))
    
    def _fit_exponential(
        self, 
        times: List[float], 
        scores: List[float]
    ) -> Tuple[float, float, float]:
        """
        最小二乘法拟合指数曲线 s = a * exp(-b * t)
        转换为线性：log(s) = log(a) - b * t
        然后用线性回归求解
        Returns: (a, b, r_squared)
        """
        valid = [(t, s) for t, s in zip(times, scores) if s > 0.01]
        
        if len(valid) < 2:
            return 1.0, 0.1, 0.0
        
        n = len(valid)
        log_scores = [math.log(s) for t, s in valid]
        ts = [t for t, s in valid]
        
        sum_t = sum(ts)
        sum_y = sum(log_scores)
        sum_ty = sum(t * y for t, y in zip(ts, log_scores))
        sum_t2 = sum(t * t for t in ts)
        
        denominator = n * sum_t2 - sum_t * sum_t
        if denominator == 0:
            return 1.0, 0.1, 0.0
        
        b_neg = (n * sum_ty - sum_t * sum_y) / denominator
        log_a = (sum_y - b_neg * sum_t) / n
        
        a = math.exp(log_a)
        b = -b_neg
        
        # 计算 R²
        y_mean = sum_y / n
        ss_tot = sum((y - y_mean) ** 2 for y in log_scores)
        y_pred = [log_a + b_neg * t for t in ts]
        ss_res = sum((y - yp) ** 2 for y, yp in zip(log_scores, y_pred))
        
        r_squared = 1 - (ss_res / ss_tot) if ss_tot > 0 else 0.0
        
        a = max(0.5, min(1.0, a))
        b = max(0.01, min(1.0, b))
        
        return a, b, r_squared
    
    def _calc_optimal_time(self, a: float, b: float) -> float:
        """计算得分降到 target_score 的时间"""
        if b <= 0 or a <= self.target_score:
            return 7.0
        
        t = -math.log(self.target_score / a) / b
        return max(0.5, min(30.0, t))
    
    def _default_curve(self) -> ForgettingCurve:
        """默认遗忘曲线（艾宾浩斯）"""
        return ForgettingCurve(a=0.95, b=0.15, optimal_interval=3.0, data_points=0, r_squared=0.0)
    
    def _default_schedule(self, last_review: datetime) -> ReviewSchedule:
        """默认复习计划"""
        return ReviewSchedule(
            next_review=last_review + timedelta(days=1),
            optimal_interval=1.0, predicted_score=0.7, confidence=0.5
        )
    
    def _get_curve(self, student_id: str, knowledge_id: str) -> Optional[ForgettingCurve]:
        """获取曲线参数（先查内存，再查数据库）"""
        if student_id in self.curves and knowledge_id in self.curves[student_id]:
            return self.curves[student_id][knowledge_id]
        
        curve = self._load_curve_from_db(student_id, knowledge_id)
        if curve:
            if student_id not in self.curves:
                self.curves[student_id] = {}
            self.curves[student_id][knowledge_id] = curve
        
        return curve
    
    def _save_curves_to_db(self, student_id: str, curves: Dict[str, ForgettingCurve]):
        """保存曲线参数到数据库"""
        for knowledge_id, curve in curves.items():
            self.db.execute("""
                INSERT INTO forgetting_curves 
                (student_id, knowledge_id, a, b, optimal_interval, data_points, r_squared)
                VALUES ($1, $2, $3, $4, $5, $6, $7)
                ON CONFLICT (student_id, knowledge_id) 
                DO UPDATE SET a=$3, b=$4, optimal_interval=$5, data_points=$6, r_squared=$7
            """, student_id, knowledge_id, curve.a, curve.b, 
                curve.optimal_interval, curve.data_points, curve.r_squared)
    
    def _load_curve_from_db(self, student_id: str, knowledge_id: str) -> Optional[ForgettingCurve]:
        """从数据库加载曲线参数"""
        result = self.db.execute("""
            SELECT a, b, optimal_interval, data_points, r_squared
            FROM forgetting_curves
            WHERE student_id = $1 AND knowledge_id = $2
        """, student_id, knowledge_id)
        
        if result:
            row = result[0]
            return ForgettingCurve(a=row[0], b=row[1], optimal_interval=row[2], data_points=row[3], r_squared=row[4])
        return None
```

### 3.3 创新三：知识脚手架动态生成

#### 3.3.1 问题定义

**现有问题：**
- 固定难度的学习材料
- 统一的提示和解释
- 没有"脚手架"概念
- 无法根据学生水平动态调整支持程度

**目标：** 根据学生水平，动态生成不同难度的学习支持（脚手架），随能力提升逐渐撤除。

#### 3.3.2 技术方案

**核心思路：** 3 级脚手架模板 + 动态调整算法

```
学生请求学习知识点 A
         │
         ▼
    查询学生当前水平
    （基于画像 + 近期表现）
         │
    ┌────┴────┐
    ▼         ▼         ▼
 初学者    进阶者    高级者
 掌握度    掌握度    掌握度
 <50%     50-80%    >80%
    │         │         │
    ▼         ▼         ▼
 高支持    中支持    低支持
 脚手架    脚手架    脚手架
    │         │         │
    ▼         ▼         ▼
 详细步骤  减少提示  只给框架
 大量提示  抽象示例  自主探索
 具体示例  适度解释  挑战性问题
 引导练习  半引导    独立练习
    │         │         │
    ▼         ▼         ▼
 掌握度提升后自动升级到下一级
```

```python
# core/scaffold_generator.py

from typing import Dict, List, Optional
from dataclasses import dataclass, field
from enum import Enum
import logging

logger = logging.getLogger(__name__)

class ScaffoldLevel(Enum):
    """脚手架级别"""
    HIGH = "high"      # 高支持（初学者）
    MEDIUM = "medium"  # 中支持（进阶者）
    LOW = "low"        # 低支持（高级者）

@dataclass
class ScaffoldConfig:
    """脚手架配置"""
    level: ScaffoldLevel
    include_steps: bool
    include_hints: bool
    example_type: str  # "concrete", "abstract", "none"
    explanation_depth: str  # "detailed", "moderate", "minimal"
    practice_type: str  # "guided", "semi_guided", "independent"
    include_challenge: bool = False

@dataclass
class ScaffoldContent:
    """脚手架内容"""
    knowledge_point: str
    level: ScaffoldLevel
    steps: Optional[List[str]] = None
    hints: Optional[List[str]] = None
    examples: Optional[List[Dict]] = None
    explanation: Optional[str] = None
    practice: Optional[Dict] = None
    challenge: Optional[Dict] = None

class ScaffoldGenerator:
    """知识脚手架动态生成器"""
    
    SCAFFOLD_CONFIGS = {
        ScaffoldLevel.HIGH: ScaffoldConfig(
            level=ScaffoldLevel.HIGH,
            include_steps=True, include_hints=True,
            example_type="concrete", explanation_depth="detailed",
            practice_type="guided", include_challenge=False
        ),
        ScaffoldLevel.MEDIUM: ScaffoldConfig(
            level=ScaffoldLevel.MEDIUM,
            include_steps=True, include_hints=False,
            example_type="abstract", explanation_depth="moderate",
            practice_type="semi_guided", include_challenge=False
        ),
        ScaffoldLevel.LOW: ScaffoldConfig(
            level=ScaffoldLevel.LOW,
            include_steps=False, include_hints=False,
            example_type="none", explanation_depth="minimal",
            practice_type="independent", include_challenge=True
        )
    }
    
    LEVEL_THRESHOLDS = {
        ScaffoldLevel.HIGH: 0.5,
        ScaffoldLevel.MEDIUM: 0.8
    }
    
    def __init__(self, llm_client):
        self.llm = llm_client
    
    def generate(self, knowledge_point: str, student_mastery: float) -> ScaffoldContent:
        """根据学生掌握度生成脚手架"""
        level = self._determine_level(student_mastery)
        config = self.SCAFFOLD_CONFIGS[level]
        
        content = ScaffoldContent(knowledge_point=knowledge_point, level=level)
        
        if config.include_steps:
            content.steps = self._generate_steps(knowledge_point, config)
        if config.include_hints:
            content.hints = self._generate_hints(knowledge_point)
        if config.example_type != "none":
            content.examples = self._generate_examples(knowledge_point, config.example_type)
        
        content.explanation = self._generate_explanation(knowledge_point, config.explanation_depth)
        content.practice = self._generate_practice(knowledge_point, config.practice_type)
        
        if config.include_challenge:
            content.challenge = self._generate_challenge(knowledge_point)
        
        logger.info(f"为知识点 '{knowledge_point}' 生成 {level.value} 级别脚手架 (掌握度={student_mastery:.2f})")
        
        return content
    
    def adjust_level(
        self, 
        current_level: ScaffoldLevel,
        recent_performance: float,
        trend: str
    ) -> ScaffoldLevel:
        """根据表现动态调整脚手架级别"""
        if recent_performance > 0.8 and trend == "improving":
            return self._upgrade_level(current_level)
        if recent_performance < 0.5 or trend == "declining":
            return self._downgrade_level(current_level)
        return current_level
    
    def _determine_level(self, mastery: float) -> ScaffoldLevel:
        if mastery < self.LEVEL_THRESHOLDS[ScaffoldLevel.HIGH]:
            return ScaffoldLevel.HIGH
        elif mastery < self.LEVEL_THRESHOLDS[ScaffoldLevel.MEDIUM]:
            return ScaffoldLevel.MEDIUM
        else:
            return ScaffoldLevel.LOW
    
    def _upgrade_level(self, current: ScaffoldLevel) -> ScaffoldLevel:
        order = [ScaffoldLevel.HIGH, ScaffoldLevel.MEDIUM, ScaffoldLevel.LOW]
        idx = order.index(current)
        return order[idx + 1] if idx < len(order) - 1 else current
    
    def _downgrade_level(self, current: ScaffoldLevel) -> ScaffoldLevel:
        order = [ScaffoldLevel.HIGH, ScaffoldLevel.MEDIUM, ScaffoldLevel.LOW]
        idx = order.index(current)
        return order[idx - 1] if idx > 0 else current
    
    def _generate_steps(self, knowledge: str, config: ScaffoldConfig) -> List[str]:
        if config.level == ScaffoldLevel.HIGH:
            return [
                f"步骤 1：阅读{knowledge}的核心概念定义",
                f"步骤 2：查看下面的具体示例，理解概念的应用",
                f"步骤 3：按照提示完成引导式练习",
                f"步骤 4：独立完成类似题目，检验理解程度"
            ]
        else:
            return [
                f"步骤 1：理解{knowledge}的核心概念",
                f"步骤 2：查看抽象示例，思考应用场景",
                f"步骤 3：完成练习题"
            ]
    
    def _generate_hints(self, knowledge: str) -> List[str]:
        return [
            f"提示：注意{knowledge}的关键公式推导过程",
            f"提示：可以参考下面的具体示例",
            f"提示：如果遇到困难，回顾步骤 1 的核心概念"
        ]
    
    def _generate_examples(self, knowledge: str, example_type: str) -> List[Dict]:
        if example_type == "concrete":
            return [{"title": f"{knowledge}的具体示例", "content": "假设我们要解决一个具体问题...", "difficulty": "easy"}]
        else:
            return [{"title": f"{knowledge}的抽象示例", "content": "考虑一般情况下的{knowledge}...", "difficulty": "medium"}]
    
    def _generate_explanation(self, knowledge: str, depth: str) -> str:
        if depth == "detailed":
            return f"{knowledge}是本课程的重要概念。它指的是...（详细解释）。理解这个概念对后续学习至关重要。"
        elif depth == "moderate":
            return f"{knowledge}指的是...（适度解释）。"
        else:
            return f"关于{knowledge}，请参考教材第 X 章。"
    
    def _generate_practice(self, knowledge: str, practice_type: str) -> Dict:
        if practice_type == "guided":
            return {"type": "guided", "description": f"按照提示完成{knowledge}的练习", "hints_available": True}
        elif practice_type == "semi_guided":
            return {"type": "semi_guided", "description": f"完成{knowledge}的练习", "hints_available": False}
        else:
            return {"type": "independent", "description": f"独立完成{knowledge}的综合练习", "hints_available": False}
    
    def _generate_challenge(self, knowledge: str) -> Dict:
        return {
            "question": f"请自主探索{knowledge}在实际项目中的应用",
            "requirement": "写出至少 3 个实际应用场景，并分析优缺点",
            "evaluation_criteria": ["应用场景的合理性", "分析的深度", "创新性"]
        }
```

```python
# core/cross_modal_linker.py

from neo4j import GraphDatabase

class CrossModalLinker:
    """跨模态知识关联器"""
    
    def __init__(self, neo4j_uri, user, password):
        self.driver = GraphDatabase.driver(neo4j_uri, auth=(user, password))
    
    def link_resources(self, knowledge_id: str, resources: list):
        """将不同模态的资源关联到同一知识点"""
        with self.driver.session() as session:
            for res in resources:
                session.run("""
                    MATCH (k:Knowledge {id: $kid})
                    MERGE (r:Resource {id: $rid, type: $type})
                    SET r.url = $url, r.metadata = $metadata
                    MERGE (k)-[:HAS_RESOURCE {modal: $modal}]->(r)
                """, kid=knowledge_id, rid=res['id'], type=res['type'],
                    url=res['url'], metadata=res.get('metadata', {}),
                    modal=res.get('modal', 'text'))
    
    def get_related_resources(self, knowledge_id: str, current_type: str) -> list:
        """获取关联的其他模态资源"""
        with self.driver.session() as session:
            result = session.run("""
                MATCH (k:Knowledge {id: $kid})-[:HAS_RESOURCE]->(r:Resource)
                WHERE r.type <> $current_type
                RETURN r.id AS id, r.type AS type, r.url AS url, r.metadata AS metadata
            """, kid=knowledge_id, current_type=current_type)
            return [dict(record) for record in result]
    
    def get_timestamp_link(self, video_id: str, timestamp: float) -> dict:
        """根据视频时间戳获取关联的文档段落"""
        with self.driver.session() as session:
            result = session.run("""
                MATCH (v:Resource {id: $vid, type: 'video'})<-[:HAS_RESOURCE]-(k:Knowledge)
                -[:HAS_RESOURCE]->(d:Resource {type: 'document'})
                WHERE $timestamp >= v.start_time AND $timestamp <= v.end_time
                RETURN d.id AS doc_id, d.url AS doc_url
            """, vid=video_id, timestamp=timestamp)
            record = result.single()
            return dict(record) if record else None
```

```python
# core/group_intelligence_optimizer.py

from collections import defaultdict

class GroupIntelligenceOptimizer:
    """群体智能优化器"""
    
    def __init__(self, db):
        self.db = db
    
    def analyze_patterns(self, min_support: int = 100) -> list:
        """分析群体学习模式"""
        all_sequences = self.db.get_all_learning_sequences()
        
        patterns = []
        pair_stats = defaultdict(lambda: {'total': 0, 'success': 0})
        
        for seq in all_sequences:
            for i in range(len(seq) - 1):
                A, B = seq[i]['knowledge_id'], seq[i+1]['knowledge_id']
                pair_stats[(A, B)]['total'] += 1
                if seq[i+1]['score'] > 0.7:
                    pair_stats[(A, B)]['success'] += 1
        
        for (A, B), stats in pair_stats.items():
            if stats['total'] >= min_support:
                success_rate = stats['success'] / stats['total']
                if success_rate > 0.75:
                    patterns.append({
                        'from': A, 'to': B,
                        'success_rate': success_rate,
                        'support': stats['total']
                    })
        
        return sorted(patterns, key=lambda x: x['success_rate'], reverse=True)
    
    def optimize_path(self, student_id: str, candidate_path: list) -> list:
        """用群体规律优化个体路径"""
        patterns = self.db.get_high_quality_patterns()
        optimized = list(candidate_path)
        for i in range(len(optimized) - 1):
            for pattern in patterns:
                if optimized[i] == pattern['to'] and optimized[i+1] == pattern['from']:
                    optimized[i], optimized[i+1] = optimized[i+1], optimized[i]
                    break
        return optimized
```

### 3.6 创新六：认知负荷动态调节 (P1)

#### 3.6.1 问题定义
**现有问题：** 固定难度分级，无法实时监测学生的认知负荷。
**目标：** 通过交互行为实时估算认知负荷，动态调整内容难度。

#### 3.6.2 技术方案
```

         │
         ▼
    估算认知负荷 (0-1)
    load = 0.4*错误率 + 0.3*响应时间 + 0.3*提示次数
         │
         ▼

    load > 0.7 → 降低难度
    load < 0.3 → 提高难度
```

```python
# core/cognitive_load_estimator.py

class CognitiveLoadEstimator:
    """认知负荷估算器"""
    
    def estimate(self, recent_actions: list) -> float:
        """通过行为数据估算认知负荷"""
        if not recent_actions:
            return 0.5
        
        avg_time = sum(a['response_time'] for a in recent_actions) / len(recent_actions)
        error_rate = sum(1 for a in recent_actions if a['is_error']) / len(recent_actions)
        hint_count = sum(a['hint_requests'] for a in recent_actions) / len(recent_actions)
        
        load = (
            0.4 * min(error_rate, 1.0) +
            0.3 * min(avg_time / 300, 1.0) +
            0.3 * min(hint_count / 5, 1.0)
        )
        
        return max(0.0, min(1.0, load))
    
    def adjust_difficulty(self, load: float, current_difficulty: int) -> int:
        """根据认知负荷调整难度 (1-5 级)"""
        if load > 0.7:
            return max(current_difficulty - 1, 1)
        elif load < 0.3:
            return min(current_difficulty + 1, 5)
        return current_difficulty
```

### 3.8 创新八：知识迁移规律发现 (P2)

#### 3.8.1 问题定义
**现有问题：** 知识图谱只有"前置/后续"关系，没有"迁移强度"。
**目标：** 量化"学会 A 对学 B 的帮助有多大"，发现知识迁移规律。

#### 3.8.2 技术方案
```
收集所有学生学习数据
         │
         ▼
    对每对知识点 (A, B) 分组：
    - 组 1：学过 A 的学生
    - 组 2：没学过 A 的学生
         │
         ▼
    比较两组学 B 的速度和成功率
         │
         ▼
    计算迁移效应 = (速度_有 A - 速度_无 A) / 速度_无 A
         │
         ▼
    发现显著正迁移规律 (效应 > 20%)
```

#### 3.8.3 核心代码实现
```python
# core/knowledge_transfer_analyzer.py

from collections import defaultdict

class KnowledgeTransferAnalyzer:
    """知识迁移规律分析器"""
    
    def __init__(self, db):
        self.db = db
    
    def analyze_transfer_effects(self, all_students_data: list) -> list:
        """分析知识迁移效应"""
        all_knowledge = set()
        for student in all_students_data:
            all_knowledge.update(student['learned_knowledge'])
        
        transfer_rules = []
        
        for A in all_knowledge:
            for B in all_knowledge:
                if A == B:
                    continue
                
                # 分组
                group_with_A = [s for s in all_students_data if A in s['learned_knowledge']]
                group_without_A = [s for s in all_students_data if A not in s['learned_knowledge']]
                
                if len(group_with_A) < 50 or len(group_without_A) < 50:
                    continue  # 样本不足
                
                # 计算学 B 的平均时间
                speed_with_A = sum(s['learning_time'][B] for s in group_with_A if B in s['learning_time']) / len(group_with_A)
                speed_without_A = sum(s['learning_time'][B] for s in group_without_A if B in s['learning_time']) / len(group_without_A)
                
                if speed_without_A == 0:
                    continue
                
                # 迁移效应
                transfer_effect = (speed_without_A - speed_with_A) / speed_without_A
                
                if transfer_effect > 0.2:  # 显著正迁移
                    transfer_rules.append({
                        'from': A, 'to': B,
                        'effect': transfer_effect,
                        'sample_size': len(group_with_A)
                    })
        
        return sorted(transfer_rules, key=lambda x: x['effect'], reverse=True)
```

```python
class PeerLearningMatcher:
    """智能同伴匹配器"""
    
    def find_peers(self, student_id: str, top_k: int = 5) -> list:
        """寻找学习伙伴"""
        student = self.db.get_profile(student_id)
        all_students = self.db.get_all_profiles()
        
        candidates = []
        for other in all_students:
            if other.id == student_id:
                continue
            
            # 基础互补
            complement_score = self.calc_complement(student, other)
            # 风格互补
            style_score = self.calc_style_complement(student, other)
            # 目标一致
            goal_score = self.calc_goal_similarity(student, other)
            
            total_score = 0.4 * complement_score + 0.3 * style_score + 0.3 * goal_score
            candidates.append((other, total_score))
        
        candidates.sort(key=lambda x: x[1], reverse=True)
        return [c[0] for c in candidates[:top_k]]
```

```python
class PathAgent:
    def plan(self, student_id: str, target: str, profile: dict) -> dict:
        # 1. 从知识图谱获取 DAG
        dag = self.graph.get_subgraph(target, max_depth=5)

        path = self._dijkstra(weighted_dag, start="current", end=target)

        time_estimates = self._estimate_time(path, profile)
        
        return {
            "type": "path",
            "nodes": path,
            "total_nodes": len(path),
            "estimated_weeks": sum(time_estimates) / 7,
            "explanation": self.explainable_recommender.explain_path(...)
```

```python
class ReviewAgent:
    def review(self, content: str, references: list) -> dict:
        issues = []
        
        # 第 1 道：知识检索验证
        if not self._has_reference(content, references):
            issues.append({"type": "unverified", "severity": "warning"})
        
        # 第 2 道：逻辑一致性检查
        if not self._check_logic(content):
            issues.append({"type": "logic_error", "severity": "error"})
        
        # 第 3 道：事实性核查
        if not self._check_facts(content):
            issues.append({"type": "fact_error", "severity": "error"})
        
        # 第 4 道：敏感内容检测
        if self._check_sensitive(content):
            issues.append({"type": "sensitive", "severity": "critical"})
        
        passed = not any(i["severity"] in ["error", "critical"] for i in issues)
        
        return {
            "passed": passed,
            "issues": issues,
            "quality_score": self._calc_quality_score(issues)
        }
```

#### 4.2.11 Retrieval Agent（知识检索）

**职责：** 知识库检索与增强（双引擎）

**输入：** 查询文本

**输出：** 相关知识片段

**核心逻辑：**

```python
class RetrievalAgent:
    def search(self, query: str, top_k: int = 5) -> list:
        # 引擎 1：向量检索
        vector_results = self.vector_search(query, top_k)
        
        # 引擎 2：图谱检索
        graph_results = self.graph_search(query, top_k)
        
        # RRF 融合
        merged = self.rrf_merge(vector_results, graph_results, k=60)
        
        return merged[:top_k]
    
    def vector_search(self, query: str, top_k: int) -> list:
        query_vector = self.embedder.encode(query)
        return self.milvus.search(query_vector, top_k)
    
    def graph_search(self, query: str, top_k: int) -> list:
        entities = self.extract_entities(query)
        return self.neo4j.query_entities(entities, top_k)
    
    def rrf_merge(self, vector_results: list, graph_results: list, k: int = 60) -> list:
        scores = {}
        for rank, item in enumerate(vector_results):
            key = item["id"]
            scores[key] = scores.get(key, 0) + 1.0 / (k + rank + 1)
        for rank, item in enumerate(graph_results):
            key = item["id"]
            scores[key] = scores.get(key, 0) + 1.0 / (k + rank + 1)
        
        sorted_items = sorted(scores.items(), key=lambda x: x[1], reverse=True)
        return [item for item, score in sorted_items]
```

#### 7.1.2 Chunk 切分策略

```python
def chunk_document(text: str, chunk_size: int = 512, overlap: int = 50) -> list:
    """按语义切分文档"""
    # 1. 按段落切分
    paragraphs = text.split('\n\n')
    
    chunks = []
    current_chunk = ""
    
    for para in paragraphs:
        if len(current_chunk) + len(para) > chunk_size:
            if current_chunk:
                chunks.append(current_chunk)
            # 保留重叠部分
            current_chunk = current_chunk[-overlap:] + para
        else:
            current_chunk += para
    
    if current_chunk:
        chunks.append(current_chunk)
    
    return chunks
```

### 7.3 学习画像构建

#### 7.3.1 EWMA 动态更新算法

```python
def ewma_update(old_value: float, new_value: float, alpha: float = 0.3) -> float:
    """
    EWMA 动态更新公式
    V_new = α × V_observed + (1-α) × V_old
    """
    return alpha * new_value + (1 - alpha) * old_value
```

### 7.4 学习路径规划算法

#### 7.4.1 改进 Dijkstra 算法

```python
def improved_dijkstra(dag: dict, start: str, end: str, profile: dict) -> list:
    """
    改进 Dijkstra 算法
    根据学生画像调整节点权重
    """
    import heapq
    
    # 初始化
    distances = {node: float('inf') for node in dag}
    distances[start] = 0
    previous = {node: None for node in dag}
    pq = [(0, start)]
    
    while pq:
        current_dist, current_node = heapq.heappop(pq)
        
        if current_node == end:
            break
        
        for neighbor, base_weight in dag[current_node].items():
            # 根据画像调整权重
            mastery = profile.get(neighbor, 0.5)
            adjusted_weight = base_weight * (1.5 - mastery)  # 掌握度越高，权重越低
            
            distance = current_dist + adjusted_weight
            
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                previous[neighbor] = current_node
                heapq.heappush(pq, (distance, neighbor))
    
    # 回溯路径
    path = []
    node = end
    while node is not None:
        path.append(node)
        node = previous[node]
    
    return path[::-1]
```
