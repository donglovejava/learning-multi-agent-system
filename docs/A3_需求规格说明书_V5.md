## 1 我们做什么

### 1.1 一句话

**我们构建了一个由 11 个 AI 智能体协同工作的学习系统——它通过对话了解学生，30 秒内生成 6 种个性化学习资料，并持续跟踪学习效果、动态调整学习路径。**

### 1.2 三个数字说清楚

| 数字 | 含义 | 对比传统方式 |
|------|------|-------------|
| **5 分钟** | 通过对话完成学生画像构建（6 个维度） | 传统问卷 20 分钟，且学生敷衍填写 |
| **30 秒** | 一次请求并行生成 6 种学习资料 | 教师手工备课 2-4 小时/种资源 |
| **10 秒** | 检测到学习异常后自动调整路径 | 传统系统无动态调整能力 |

### 1.3 一个完整场景

**学生小李，计算机专业大二，想学"Transformer 注意力机制"：**

```
第 1 步：小李登录系统，和 AI 聊了 5 分钟
        → 系统了解：计算机大二、目标考研、线性代数不错、概率论薄弱、视觉型学习者

第 2 步：小李说"我想学 Transformer 的注意力机制"
        → 系统 30 秒内并行生成：
           ✅ 2000 字讲解文档（难度适配他的水平）
           ✅ 10 道练习题（4 基础 + 4 中等 + 2 提高）
           ✅ 知识体系思维导图（可视化呈现）
           ✅ 90 秒动画视频（SeeDance 生成）
           ✅ Python 代码案例（可运行的注意力机制实现）
           ✅ 拓展阅读材料（相关论文和博客）

第 3 步：系统生成 8 周学习路径
        → 基于知识图谱 DAG，结合他的画像
        → 先补概率论基础 → 再学注意力机制 → 最后到 BERT/GPT

第 4 步：小李学了一周，系统在"决策树"章节发现他得分 55%
        → 自动触发预警
        → 生成补充讲解 + 3 道针对性练习
        → 调整后续路径，增加复习节点
        → 更新画像：决策树掌握度从 70% 降到 55%
```

### 1.4 我们和现有方案的本质区别

| 维度 | 现有方案 | 我们 |
|------|---------|------|
| **了解学生** | 填表/问卷（静态、敷衍） | 对话式画像（动态、自然） |
| **生成资源** | 教师手工（慢、单一） | 11 Agent 并行（快、6 种多模态） |
| **学习路径** | 固定大纲（一刀切） | DAG + 画像驱动（因人而异、实时调整） |
| **内容质量** | 依赖个人水平 | 5 层防幻觉 + 人工审核（标准化） |
| **效果跟踪** | 期末考试成绩（事后） | 10 维度实时监控（事前预警） |

### 1.5 项目背景与痛点

**痛点 1：资源同质化**
- 统一教材无法满足不同基础的学生
- 好学生"吃不饱"，差学生"跟不上"
- 教育部数据：全国高校师生比 1:17，个性化教学难以落地

**痛点 2：教师负担重**
- 教师每周花 8-12 小时准备教学材料
- 60% 时间用于重复性资源整理和改编
- 高质量个性化资源（分层练习、思维导图、教学视频）生产成本极高

**痛点 3：学习路径僵化**
- 75% 学生认为学习路径"不够灵活"
- 60% 学生希望系统能"根据我的情况自动调整"
- 传统 LMS 只有固定课程大纲，无法因材施教

**痛点 4：AI 生成内容不可信**
- 大模型幻觉问题严重，教育场景零容忍
- 现有 AI 教育产品缺乏有效的内容质量保障机制
- 教师不敢直接使用 AI 生成内容

### 1.6 术语定义

| 术语 | 定义 | 本系统中的应用 |
|------|------|---------------|
| **Agent（智能体）** | LLM + 工具 + 记忆 + 自主决策能力的程序实体 | 11 个专职 Agent 各司其职 |
| **多智能体系统** | 多个 Agent 通过编排框架协同工作的架构 | LangGraph 编排 11 个 Agent |
| **学习画像** | 基于多维度数据构建的学生特征模型 | 6 维度动态画像（EWMA 算法更新） |
| **RAG** | 检索增强生成，先检索知识再生成内容 | Document/Quiz Agent 的核心工作流 |
| **知识图谱** | 以图结构组织知识实体及关系的语义网络 | Neo4j 存储知识点前置/后续关系 |
| **DAG** | 有向无环图，建模知识点依赖关系 | Path Agent 的学习路径建模基础 |
| **EWMA** | 指数加权移动平均算法 | 画像维度值的动态更新公式 |
| **防幻觉** | 防止 AI 生成错误内容的多层检测机制 | 5 层防线保障内容准确率>98% |
| **SeeDance** | 科大讯飞多模态生成大模型 | Video Agent 调用生成教学视频 |
| **RRF** | Reciprocal Rank Fusion 融合排序算法 | 向量检索 + 图谱检索结果融合 |

## 3 需求规定

### 3.1 功能需求

#### 3.1.1 对话式学习画像构建模块

**6 个画像维度：**

| 维度 | 数据来源 | 评估方式 |
|------|----------|----------|
| 专业背景 | 对话抽取 | NLP 实体识别 |
| 学习目标 | 对话抽取 | 分类识别（考研/期末/竞赛/兴趣/工作） |
| 知识基础 | 评估问题 + 答题 | 知识图谱关联评估 |
| 学习风格 | 对话 + 行为数据 | 视觉/听觉/动觉/读写分类 |
| 学习动机 | 对话 + 行为数据 | 内在/外在动机识别 |
| 元认知能力 | 评估问题 | 自我监控/自我调节评估 |

**核心功能需求：**

| FR 编号 | 功能名称 | 优先级 | 验收标准 |
|---------|----------|--------|----------|
| FR-001 | 对话开场与引导 | P0 | 开场白 1 秒内显示，包含目的说明 |
| FR-002 | 专业信息抽取 | P0 | 专业匹配准确率>90% |
| FR-003 | 学习目标识别 | P0 | 区分 5 种以上目标类型 |
| FR-004 | 知识基础评估 | P0 | 评估结果偏差<20% |
| FR-005 | 学习风格识别 | P1 | 识别 4 种以上学习风格 |
| FR-006 | 学习动机识别 | P1 | 区分内在/外在动机 |
| FR-007 | 元认知能力评估 | P1 | 评估维度完整，提供改进建议 |
| FR-008 | 6 维度画像计算 | P0 | EWMA 算法计算，置信度准确 |
| FR-009 | 画像摘要生成 | P0 | 摘要准确反映画像，语言友好 |

#### 3.1.3 个性化学习路径规划模块

**功能概述：** 基于 DAG 和改进 Dijkstra 算法，结合学生画像动态规划学习路径。

**核心功能需求：**

| FR 编号 | 功能名称 | 优先级 | 验收标准 |
|---------|----------|--------|----------|
| FR-025 | DAG 知识图谱建模 | P0 | 无环检测通过，节点关系正确 |

### 5.2 核心创新（P0）—— 必须实现

#### IN-03：可解释 AI 决策

**一句话：** 每个推荐都有理由，学生知道为什么学这个。

**现有问题：**
- AI 推荐学习内容，但不解释为什么
- 学生不知道为什么要学这个，学习动机不足
- 路径调整没有理由，学生不理解

**我们的创新：**
- 每个推荐都有**解释**："推荐你先学线性代数，因为你的概率论基础薄弱（掌握度 45%），而线性代数是概率论的前置知识"
- 路径调整有**理由**："检测到你在决策树章节得分 55%，所以增加了 3 道针对性练习"
- 资源生成有**依据**："这份文档的难度适配你的水平，因为你之前的练习正确率是 75%"

**技术实现：**
- **知识推理：** 查询知识图谱的关系链（线性代数 → 前置知识 → 概率论 → 前置知识 → Transformer）
- **自然语言生成：** 将推理链转化为学生可理解的解释
- **决策追溯：** 记录每个决策的完整推理链，学生可以查看"为什么推荐这个"

**为什么是核心创新：**
- AI 可解释性是 2026 年的热点，教育场景的完整实现是新的
- 技术难度适中（知识推理 + NLG），20 天可以做原型
- 评委容易理解，演示效果好
- 提升用户信任度和学习动机

**20 天实现计划：**
- 第 1-3 天：设计知识推理算法
- 第 4-7 天：实现解释生成模块
- 第 8-10 天：集成到推荐系统
- 第 11-15 天：前端展示和交互优化
- 第 16-20 天：测试和演示准备

#### IN-04：个性化遗忘曲线

**一句话：** 根据每个学生的实际数据，建立专属遗忘曲线，在最佳时间点推送复习。

**现有问题：**
- 统一复习计划（1 天后、3 天后、7 天后）
- 不考虑个体差异（有人记得快，有人忘得快）
- 不考虑知识特性（概念性知识 vs 程序性知识遗忘速度不同）

**我们的创新：**
- 根据每个学生的**实际测试数据**，拟合个性化遗忘曲线：
  - 学生 A：学完后 1 天遗忘 20%，3 天遗忘 40%
  - 学生 B：学完后 1 天遗忘 40%，3 天遗忘 70%
- 在**最佳时间点**推送复习：
  - 学生 A：3 天后复习
  - 学生 B：1 天后复习

**技术实现：**
- **数据收集：** 记录每次学习时间和测试得分
- **曲线拟合：** 用指数衰减模型 score = a × exp(-b × time) 拟合
- **最优复习时间：** 计算得分降到 70% 的时间点
- **复习调度：** 根据遗忘曲线生成个性化复习计划

**为什么是核心创新：**
- 教育理论支撑强（艾宾浩斯遗忘曲线）
- 技术难度适中（曲线拟合算法），20 天可以做原型
- 用户价值明显（提高记忆效率）
- 可以用模拟数据演示，不需要真实长期数据

**20 天实现计划：**
- 第 1-3 天：设计数据收集方案
- 第 4-7 天：实现曲线拟合算法
- 第 8-12 天：实现复习调度模块
- 第 13-17 天：前端展示（遗忘曲线图表）
- 第 18-20 天：用模拟数据演示

#### IN-07：知识脚手架动态生成

**一句话：** 根据学生水平，动态生成不同难度的学习支持（脚手架），随能力提升逐渐撤除。

**现有问题：**
- 固定难度的学习材料
- 统一的提示和解释
- 没有"脚手架"概念

**我们的创新：**
- 根据学生当前水平，**动态生成学习脚手架**：
  - **初学者：** 详细的步骤分解、大量提示、具体示例
  - **进阶者：** 减少提示、增加抽象概念、要求自主推理
  - **高级者：** 只给问题框架、要求自主探索
- 脚手架**随能力提升自动撤除**

**技术实现：**
- **水平评估：** 根据画像和近期表现判断学生水平
- **脚手架策略：** 设计 3 级脚手架（高/中/低支持）
- **动态调整：** 根据学习表现自动调整脚手架级别
- **撤除机制：** 当学生掌握度>80% 时，逐渐减少支持

**为什么是核心创新：**
- 教育理论经典（维果茨基最近发展区 + 脚手架理论）
- 技术难度适中（规则引擎 + 动态调整），20 天可以实现
- 用户体验好（因材施教的直观体现）
- 演示效果明显（同一知识点的 3 种版本对比）

**20 天实现计划：**
- 第 1-3 天：设计 3 级脚手架模板
- 第 4-8 天：实现脚手架生成模块
- 第 9-12 天：实现动态调整算法
- 第 13-17 天：前端展示（脚手架级别切换）
- 第 18-20 天：准备对比演示

### 5.3 重要创新（P1）—— 应该实现

#### IN-01：跨模态知识关联

**一句话：** 文档/视频/题目都关联到同一个知识点，学一个自动推其他。

**创新价值：** 现有资源是孤岛，我们打通了跨模态关联。

**技术难度：** 高（需要跨模态内容对齐）

**20 天可行性：** ⚠️ 可以做简化版（手动标注关联关系）

**建议：** 如果时间允许，实现核心功能（文档↔题目关联），视频关联可以简化。

---

#### IN-02：群体智能优化

**一句话：** 用所有学生的学习数据，发现群体规律，优化每个学生的学习路径。

**创新价值：** 从"个体学习"变成"群体智慧"。

**技术难度：** 高（需要因果推断 + 隐私保护）

**20 天可行性：** ⚠️ 可以做简化版（用统计分析代替因果推断）

**建议：** 用模拟数据演示群体规律发现，真实数据收集可以后续迭代。

---

#### IN-05：认知负荷动态调节

**一句话：** 通过交互行为实时估算认知负荷，动态调整内容难度。

**创新价值：** 从"固定难度"变成"动态适应"。

**技术难度：** 中（需要建立行为数据→认知负荷的映射）

**20 天可行性：** ⚠️ 可以做简化版（用启发式规则代替复杂模型）

**建议：** 用规则引擎实现（答题速度 + 错误率→难度调整），ML 模型可以后续迭代。

---

#### IN-08：元认知能力培养

**一句话：** 显性培养学生的"学会学习"能力（预测→反思→校准）。

**创新价值：** 从"只教知识"变成"教学习方法"。

**技术难度：** 中（需要设计反思机制和校准训练）

**20 天可行性：** ✅ 可以做原型

**建议：** 实现核心功能（学习前预测 + 学习后对比），长期跟踪可以后续迭代。

### 5.4 支撑创新（P2）—— 有时间就做

#### IN-06：知识迁移规律发现

**一句话：** 发现"学会 A 对学 B 有帮助"的规律，量化知识迁移效应。

**创新价值：** 量化知识之间的迁移效应，优化学习路径。

**不推荐原因：** 需要大规模群体数据才能发现规律，20 天无法积累足够数据。

**创新价值：** 从"追求简单"变成"必要难度"。

**不推荐原因：** 教育理论较新，评委可能不理解；实现难度不低但收益不明显。

**不推荐原因：** 需要长期跟踪才能看到效果，20 天无法验证；实现复杂但收益不明显。

### 5.5 不推荐创新（P3）—— 本次比赛不做

#### IN-12：智能同伴匹配 ❌

**一句话：** 根据基础/风格/目标匹配学习伙伴，实现协作学习。

**不推荐理由：**
1. **需要真实用户：** 20 天无法积累足够用户进行匹配
2. **功能复杂：** 需要设计匹配算法 + 协作界面 + 通信机制
3. **收益不明显：** 在线教育的同伴学习效果未经充分验证
4. **风险高：** 如果匹配效果不好，反而影响用户体验

**建议：** 可以在 V2 版本实现，V1 专注个体学习体验。

---

#### IN-13：多模态输入自适应 ❌

**一句话：** 支持文本/语音/图片/手写多种输入方式。

**不推荐理由：**
1. **技术集成复杂：** 需要集成 OCR、语音识别、手写识别等多种技术
2. **讯飞 API 限制：** 部分识别 API 可能不在免费额度内
3. **收益不明显：** 文本输入已经满足核心需求
4. **分散精力：** 20 天应该专注核心功能，不要分散到输入方式

**建议：** V1 只用文本输入，V2 再考虑语音和图片输入。
